# 开发日志 2026-04-08

## 完成的工作

- 新增 `benchmarks/benchmark_inference.py`，统一测 CPU / CUDA / CUDA+CPU-offload
- 新增 `benchmarks/pim_microbench/`，包含 UPMEM host benchmark、DPU kernel、build/run 脚本
- 在真实 Qwen3 checkpoint 上跑出 CPU inference benchmark
- 在 simulator 模式下跑通 PIM microbenchmark

## 关键发现

- 当前会话的 NVIDIA 内核模块已加载，但 `/dev/nvidia*` 设备节点不存在，所以 `torch.cuda.is_available()` 仍为 `False`
- `dpu-diag` 能读取大量 rank 的 MCU 信息，但 `dpu_alloc_ranks` 失败，说明硬件枚举和用户态分配不是一回事
- UPMEM simulator 需要 profile 与编译目标芯片版本对齐；当前可用 `backend=simulator,chipId=0x42`
- 进一步确认当前 Codex 执行会话把 `/dev` 挂成了私有 tmpfs，因此宿主机 GPU/DPU 字符设备被整体屏蔽

## 实测结果

- inference benchmark 结果文件：`benchmarks/results/inference_2026-04-08.json`
- PIM simulator benchmark 结果文件：`benchmarks/results/pim_simulator_2026-04-08.json`
- PIM 硬件分配失败日志：`benchmarks/results/pim_hw_2026-04-08.stderr`
- 用户宿主机补充结果：真实 PIM 硬件 benchmark 已成功，1 rank / 64 DPUs / 1 MiB per DPU / 2 repetitions 条件下，host->PIM 约 `2.62 GB/s`，PIM->host 约 `1.51 GB/s`，kernel 有效带宽约 `11.89 GB/s`，端到端约 `92.65 ms`
- 用户宿主机补充结果：真实 CUDA 设备节点存在，但 `backend=cuda` 对 Qwen3-30B-A3B 仍 OOM，因此需要继续看 `cuda_cpu_offload`
