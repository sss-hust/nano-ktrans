# 开发日志 2026-04-09

## 完成的工作

- 为 `Qwen3` 增加基于 safetensor 键名的 expert 布局自适应
- 修复 `cuda_cpu_offload` 的 CPU fallback 内存翻倍问题
- 修复 PyTorch attention fallback 中 causal mask 的 dtype 不匹配问题
- 在真机上跑通 `Qwen3-30B-A3B + cuda_cpu_offload`
- 重跑并清洗 PIM 硬件 benchmark，拿到 1/4/8/16/32 rank 结果
- 将 `ExpertOffloadBackend` / `PIMMoEBackend` 接入 `HybridMoE` 主链路，并新增 `cuda_pim_shadow` benchmark

## 关键发现

- 当前 Qwen3 checkpoint 实际采用 unpacked experts，不是 `gate_up_proj` packed layout
- `cuda_cpu_offload` 现在能稳定运行，但 CPU offload 仍然是纯 PyTorch fallback
- 当前 PIM microbenchmark 测的是整数 affine transform 和带宽，不是浮点算力
- `pim_shadow` 已能在真实推理主链路中运行并收集 per-layer offload 统计，但数值仍由 CPU fallback 产生
